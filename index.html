Biblioteca Escolar Digital

Requisitos Funcionais Requisitos Não Funcionais Requisitos Inversos

Requisitos Funcionais

  • O sistema deve notificar os usuários sobre a necessidade de devolução de livros.
  • O sistema deve permitir que professores recomendem livros para os estudantes.
  • O sistema deve permitir visualizar exemplares disponíveis para cada livro.
  • O sistema deve permitir a reserva de livros com antecedência.
  • O sistema deve realizar o levantamento dos livros mais lidos pelos alunos.
  • O sistema deve conter cópias digitais dos exemplares para leitura online.
  • O sistema deve acompanhar as leituras digitais feitas pelos alunos.
  • O sistema deve fornecer relatórios de leitura apenas para funcionários autorizados.
  • O sistema deve conter um fórum literário para que os alunos discutam os livros lidos.
  • O sistema deve permitir o anexo de atividades personalizadas associadas aos livros lidos.
  • O sistema deve recomendar leituras com base nas leituras e avaliações anteriores de cada aluno.
  • O sistema deve organizar os livros por tags como: gênero, autor, tema.

Requisitos Não Funcionais

  • O sistema deve garantir segurança e controle de acesso para diferentes perfis (aluno, professor, funcionário).
  • O sistema deve garantir a confidencialidade das informações dos usuários.
  • O sistema deve ser acessível via diferentes dispositivos (computadores, tablets, celulares).
  • O sistema deve ter uma interface intuitiva e amigável para facilitar o uso.
  • O sistema deve ser eficiente, com bom tempo de resposta para leitura digital e reservas.
  • O sistema deve estar disponível continuamente, com alta disponibilidade e manutenção programada.

Requisitos Inversos

  • O sistema não deve permitir a entrada de usuários não autorizados.
  • O sistema não deve permitir o cadastro de usuários não autorizados.
  • O sistema não deve disponibilizar relatórios de leitura digital para usuários não autorizados.
